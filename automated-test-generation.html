Automated Test Generation

Automated test generation is always what people tend to as writing test cases manually is quite tedious. However, it is not practical for entire systems to using that method, as it will have a lot of possible input for the entire process. So normally, there are techniques for automatically testing the functionality of the smallest testable parts of an application, called units, and so this approach called unit testing.

Study Direction

As the automated test is using a machine to auto-generation all the possible input for the unit, then in order to increase the effectiveness, people need to find the way to role out the illegal and redundant input. So there mainly two methods to deal with this situation.

Korat

Korat is a project conducted by three graduates in MIT, the principle to here is utilized the pre-condition to role out the part of bad input and then using a vector to represent the instructions.(using type to decide the possible node, and using k to determine the searching space.

So it can really get the all the possible inputs however the performance of that method largely depends on the pre-condition(i.e a bad pre-condition will lead to bad Korat testing

Feedback-directed testing

Unlike Korat depended on the pre-condition, feedback-directed testing relly on the iterative get the output and using the sub output to construct the following one..
